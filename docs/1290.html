不，我不是在说爪哇树脂。我说的是IoT resin.io 。

Resin.io让在一系列设备上部署应用变得轻而易举，包括Pi、Arduino，甚至是相当强大的英特尔NUC。他们已经支持十几个设备系列，并且新的设备也在不断地被支持。

在网络上，基于云的部署已经取得了很大进展，有了诸如 Heroku (易于使用)AWS elastic beanstalk(具有CloudFormation扩展的无限能力)drie.co(安全)和 now (每次部署一个URL)。

现在想象一下上面所有的，但是使用物理翻转硬件。那就是 resin.io 。

球形产品

它。只是。作品。

  1. 用他们漂亮的蚀刻机工具烧掉你的SD卡。

2.将SD卡插入您的设备

3.打开设备(并将其连接到互联网)

4.参见 resin.io web仪表板上的装置

5.通过“git推送”部署您的软件

软件更新很容易，全舰队，快如闪电(见亲提示)。大多数事情都可以通过他们的网络仪表板来完成。对于高级用户，一切都可以通过API、CLI和 SDK 获得。

别误会，这一切都不是一夜之间发生的。预蚀刻机，从ISO烧SD卡有点麻烦。偶尔也会有麻烦，比如查看你的日志超过了仪表盘上显示的100行左右(见专业提示)。

尽管总的来说，他们在过去的几年里坚持不懈地关注他们的用户，这是显而易见的。

狂热的支持

自定义linux内核头文件编译有问题吗？他们的工程师将与你一起工作，立即支持用例(直到它完成，这是棘手的)。然后，他们会将问题添加到待办事项中进行永久修复(您可以看到！，参见下面的透明度)。然后他们会通知你最新情况。

我们提出的问题总是倾向于在几个小时内，或者最多第二天出现在文件中。毫不奇怪，我们几乎没有理由再联系支持部门了。

老实说，这已经到了我们希望到 RTFM 的地步，因为它做得太好了，他们的支持太好了，我们不会觉得我们在浪费它。

更新— 2016-10-07

按照他们狂热支持的传统，我们今天收到了一个爱心包裹(来自希腊雅典)！

专业提示

  • 更快的部署:因为它都是基于docker的，所以如果您有很大的依赖性，您可以使用您自己的映像，在docker注册表上进行预编译。比如我们用OpenCV3 做到了这一点。
  • 日志:通过管道将脚本输出到一个文件，或者尝试使用一个低级工具。
  • 图像大小:使用阿尔卑斯山为微小的图像[ 博客 ]。
