同构的Web应用 :客户端和服务器端都包含MVC。相同的渲染引擎和相同的JavaScript逻辑可以在服务器端使用(由 Mehmetcan Gayberi )。

这是React-Redux-Boilerplate的最佳实践，这是一个展示同构应用机制的简单项目，下面是一些对我来说新的和重要的概念:

客户端渲染

构建由浏览器加载的静态JS文件。

服务器渲染

用户请求一个页面，它将所需的组件呈现为一个HTML字符串，然后作为响应发送给客户端(由 Redux )。

减速器

动作描述了发生了一些事情的事实，但是没有具体说明应用程序的状态如何变化来响应。这是减速器的工作。

动作动作动作

它是将数据从您的应用程序发送到您的存储的信息store.dispatch()。

动作创建者(由Redux)

1)它是创建动作的工厂函数。
2)调用动作创建者只会产生一个动作，而不会调度它。您需要调用商店的dispatch函数来实际引起变异。
ActionCreator = (...args: any) => Action

它是一个高阶函数，组成一个调度函数，返回一个新的调度函数。
