# 信任中的错误

> 原文：<https://medium.com/hackernoon/bugs-in-trust-55bce19d9ad5>

虽然一些软件团队已经制定了昂贵的零容忍政策，但是大多数团队都试图在每个 bug 的优先级和大量其他有价值的需求之间进行平衡。根据软件的性质，绝对主义和实用主义的政策都是非常合适的——例如，我希望这个软件能让我的飞机在空中 100%的时间工作，非常感谢。但是，如果一张熊猫在草地上打滚的 gif 图片中途暂停，这并不是世界末日。对于像 Greenhouse 这样的“记录系统”软件，我们发现我们自己的容忍水平更接近于“保持飞机在空中”，而不是 gif-口吃的消费者网站，但如果一个按钮比 IE10 中它应该在的位置偏了 3 倍，我们不会停止一切。

为了明确我们如何确定工作的优先级，我们开发了一个新术语，并将其融入到我们在 Greenhouse 的日常对话中，所以我写了这篇短文，与更广泛的社区分享这个术语。虽然我们在确定某些问题的优先级时会做出权衡，但我们最近制定了一项针对我们称之为*信任漏洞*的零容忍政策。让我们来定义这个术语。

*信任漏洞*不同于其他漏洞，因为它们会导致用户:

1.  不相信提供的数据是正确的，*或*
2.  不相信所采取的行动会可靠地发生

这一类的 bug 是独特的，因为它们不仅会导致挫败感，还会损害公司和客户之间的关系。每次按下按钮或出现列表时，信任漏洞都会引起怀疑，最终导致客户流失。

赢得信任比失去信任要难得多，所以不要让*信任 bug*坐以待毙。
