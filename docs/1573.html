我试过榆树，但我不知道它周围的炒作。

这一切都始于 Charles Scalfani 发表了一系列关于函数式编程的文章。在完成他的小编之后，我真的很想尝试一下 Elm ，他在这些文章中参考了很多。

介绍

Elm 是once中的语言和框架，用来写web应用。它是React和Angular等其他客户端框架的替代方案。我真的很想学习它，因为我听说了很多关于Elm和函数式编程的好消息。

尽管如此，我和埃尔姆之间还是没有结果。

榆树闪耀的地方

Elm有一些非常棒的特性，乍一看非常吸引人。它的错误信息非常好，感觉就像有一个助手在你身边。

Elm的语法是极简。不需要括号或分号。比如你可以这样声明一个函数:addFunc a b = a + b。有人可能会争辩说，这种语法很难读懂，但这只是一个时间问题，直到人们欣赏它的简单性。

然后是 榆树架构其中 是Redux的更好版本。我的意思是，您管理应用程序或组件状态的方式是相似的，但是编写的代码更少，并且不必像在 Redux 中那样安装额外的包。(现在就有读者:Redux是受了Elm的启发，你个傻逼。)

榆树不发光的地方

让我们从小事开始:

import Html exposing (..)
import Html.App as Html
import Html.Attributes exposing (..)
import Html.Events exposing (..)

这些可能是您编写的每个Elm文件的第一行，在那里将组成一些UI。如果你想给你的HTML元素一个class，你需要从Html.Attributes中导入并公开这个函数。如果你想监听事件，你需要对Html.Events做同样的事情。我承认，这是件小事，但还是有点烦人。

处理用户输入

一个典型的web场景:用户在文本字段中键入内容，然后按enter键提交输入。应该很容易实现吧？可惜不是。这是我最后想出的代码，它处理这个小案例:

import Json.Decode as Json...onEnter msg =
  let
    tagger code =
      if code == 13 then msg else NoOp
  in
    on "keydown" (Json.map tagger keyCode)

这可不好。Json在那里做什么？这只是几行代码，但是对于这样一个小例子来说，编写它们并不有趣。在Vue中，只需将v-on:keyup.enter="doSomething"添加到输入字段中，就可以实现同样的功能。

要了解在Elm中管理选择框的值有多难，请阅读本文。除了处理用户事件和呈现项目列表之外，我并没有深入研究Elm，但是这种体验足以让我知道这个旅程将走向何方。

结论

我没有得到我所希望的“ Elmlightment ”。它想与React和Vue竞争，但你用它不如用其他两个有效率。我没有像爱上Vue一样爱上Elm。

几周前我就想分享我在Elm的经历，但是我觉得我做错了什么，我没有得到Elm。但是当安德斯·霍姆勒写下他对埃尔姆的拒绝时，我知道我不是唯一有这种感觉的人。所以最后写了这篇文章。
