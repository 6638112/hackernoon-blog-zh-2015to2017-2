反应样式guidist 5

原文：https://medium.com/hackernoon/react-styleguidist-5-bf399305e34d?source=collection_archive---------8-----------------------#2017-03-31

这是React Styleguidist最大的更新，有300次提交和四个月的工作。它包含了大量的重新思考和重写。但是大部分的改变是为了使初始配置更容易。

🍕非常感谢尤里·舍甫琴柯、安德烈·奥科涅奇尼科夫和奥列格·斯洛博德斯科伊对此次发布的帮助！🍕

以下是最显著变化的简要概述。完整列表见发行说明。

如果您不熟悉Styleguidst，请查看入门指南。

创建-反应-应用程序支持

现在，Styleguidst甚至可以在没有配置的情况下使用 create-react-app 。

它将从src/components/**/*.js加载组件。以及来自Component/Readme.md或Component/Component.md的示例文件。

更简单的webpack配置

使用新的 webpackConfig 选项:

module.exports = {
  webpackConfig: {
    module: {
      loaders: [
        // Babel loader, will use your project’s .babelrc
        {
          test: /\.jsx?$/,
          exclude: /node_modules/,
          loader: 'babel-loader',
        },
        // Other loaders that is needed for your components
        {
          test: /\.css$/,
          loader: 'style-loader!css-loader?modules',
        },
      ],
    },
  },
};

您可以重用现有的webpack配置:

module.exports = {
  webpackConfig: require('./configs/webpack.js')
};

或者Styleguidist会尝试在你的项目根目录中找到webpack.config.js并使用它。

我们还为您的网络包加载器取消了一个臭名昭著的include或exclude选项要求。

其他变化

还有更多的新选项、小功能和错误修复——见发布说明。

一些潜在的变化

  • 我们已经将我们的测试从 AVA 迁移到 Jest 并且增加了很多新的测试。
  • 我们使用AST在webpack加载器中生成代码，而不是字符串连接。
  • 为了更好的隔离(感谢jss-isolate)和更容易的主题化，我们使用 JSS 代替CSS模块进行样式化。
