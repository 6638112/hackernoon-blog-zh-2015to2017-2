将我们的脚趾浸入颤动中

今年7月早些时候， Sanket Sahu 和 Atul Ranjan 有机会与来自谷歌Flutter团队的 Seth Ladd 和 Yegor Jbanov 交谈。

我们谈到了Flutter的操作，它是如何开发的，以及它的未来。

甚至在我们回到印度后，我们与Flutter团队的会谈仍在继续。我们定期与他们谈论我们的颤振实验。

使用Flutter相对于React Native的优势

  • Flutter应用程序被编译成机器代码，它不需要像React Native中JavaScript和机器代码之间的桥梁。
  • Flutter不使用Platform OEM widgets。这增加了小部件的灵活性，定制它们也比React Native容易得多。
  • Dart 是一种类型化语言。这意味着不像在React Native中，Flutter没有使用像 TypeScript 或 ReasonML 这样的东西。
  • Flutter的工具在 VSCode 和 IntelliJ 中非常棒。
  • 谷歌对Flutter的支持要比脸书对React Native的支持好得多。

我们对颤振做了什么？

  • 我们已经创建了一个名为 Start Flutter 的开源颤振主题精选列表。
  • 就像本土市场，我们创造了一个震荡市场。
  • 我们在Flutter中重写了我们著名的 React原生入门套件！我们对结果印象深刻。😄

这个版本的应用程序比React原生版本好得多。这主要是因为颤振与原生层的相互作用比反应原生层小。此外，与React Native不同，Flutter有一个内置的UI组件库，无需导入第三方UI库，如 NativeBase 。这也是为什么Flat App的Flutter版本动画流畅很多的原因。

我们希望如何帮助社区发展？

  • 我们是Flutter的早期采用者之一，我们愿意做和React Native一样的事情。由于Flutter仍在开发中，它的构建中有许多漏洞，我们很乐意帮助填补。
  • 我们使用 Redux 作为React和Dart的实际状态管理库，我们有类似Redux的Flux实现。我们很想进行实验，为Flutter想出类似的东西。
  • 我们有一些更多的初学者工具包在脑海中，这将有助于吸引更多的开发人员。我们最近推出了 React Native Seed ，这是一款为您的基础应用提供React Native入门套件的工具，其中包含您喜欢的技术。我们可能很快就会想出一个类似的颤振工具。
  • 您可能会注意到，在Flutter中设计组件的样式不同于web。因此，我们正在考虑将类似web的CSS引入Flutter。
  • 我们还想给 BuilderX 添加对Flutter的支持，这是我们为您编写React Native的新设计工具！

扑扑太神奇了！但是…

  • Flutter使用Dart，Dart是它自己的语言，而不是另一种语言的方言。不过对于有 Java 、 C# 、 JavaScript 、 TypeScript 或者 ActionScript 经验的开发者来说应该很容易学会。Flutter团队认为Dart在Flutter中是有意义的，因为它为开发人员提供了强大和高级的工具，由非常快速的热重装/ JIT开发工作流支持，并编译成本机代码，为最终用户提供快速和可预测的性能。此外，它丰富的核心库和通用范例使它可以很快学会。
  • 如上所述，在颤振中造型是不同的。
  • 我们真的需要像Expo这样的东西来测试应用程序。希望谷歌的Flutter团队或社区能为此提出一个解决方案。

总结

Flutter是一个令人惊叹的新工具，它将帮助开发者构建高性能的跨平台应用。就像React一样，它是声明性的，因此任何React或React本地开发人员都可以轻松地迁移到Flutter。

与React Native相比，Flutter更像是一个完整的解决方案，因为在Flutter中很少有开发人员需要使用第三方库的情况。Flutter的内置UI库也对其有利。😄

如何才能入门Flutter？

颤振是开源的！所以你需要做的就是查看它的官方文档，熟悉Dart，给Flutter一个机会。不要担心陷入困境，谷歌的Flutter团队和Flutter社区会一直帮助你。😄
