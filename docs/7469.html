如何以及为什么在科技会议上发言

原文：https://medium.com/hackernoon/how-and-why-to-speak-at-tech-conferences-1d50a3f548e0?source=collection_archive---------4-----------------------#2017-10-29

在过去的几年里，我在各种会议和公司内部活动上做了很多演讲。我绝不是最有成就或最知名的科技演讲者。然而，如果我有一台时间机器，我会有一些事情告诉自己，因为我第一次对说话感兴趣。

在你开始在会议上发言之前，你应该弄清楚你为什么要发言。展示可能是一项繁重的工作，你应该确保你的目标是最让你满意的。

为什么你会说话

成为网络名人

有些人有一群Twitter关注者和YouTube浏览者，这似乎让他们很开心。通过在会议上发言来建立你的公共品牌是实现这一目标的一种方式。我对“日常工作之外最小边际努力”计划的未经检验的建议:

  1. 找一份主要的、广泛使用的开源项目的日常工作(例如Angular、React、npm、Node)。“大肆宣传”可以代替“广泛使用”或添加到“广泛使用”中，以加强效果。
  2. 在你可能被接受的最著名的会议上做一些关于你团队工作的演讲。(还有博客和栈溢出和参与GitHub问题。)
  3. 看着那些可爱的推特粉丝蜂拥而至。
  4. 得到“这是否真的让我快乐”的答案。
  5. 一旦你意识到它充满了好斗的巨魔，并且鼓励我们最肤浅的想法，就退出Twitter。

啊，最后有点跑题了。但我很确定计划应该是这样的。

建立一个当地社区

一些人从参与和建立他们当地的技术社区中获得意义和快乐。为此，我建议在聚会和地区会议上演讲。(额外学分，自己整理！)如果你喜欢指导他人，较小的活动实际上可能是更好的地方，因为人们更容易在事后找到你问你问题，你可能会一次又一次地看到同一个人。这也是一个很好的方式去了解你所在地区的公司，或者在你下一次找工作时帮你一把。

推广你的产品、公司或自己

如果你经营一家咨询公司或承包公司，在会议上发言可能是找到新业务的重要途径。如果你想和当地客户合作，你应该关注当地的事件。如果你向开发者销售产品，那么你应该尽可能在最大的活动上发言。

如果你现在的工作不是你退休后的工作，那么在合适的会议上发言会引起雇主的积极关注。

获得公开演讲的经验

如果你想成为一个更自在的公众演讲者，我建议尽可能多的在活动中演讲——我会在三个不太知名的活动中演讲，而不是一个大型活动。对我个人来说，随着时间的推移，这将有助于减少对演讲的焦虑。

为你的雇主招聘

如果你看起来是一个聪明有趣的人，那么就更容易招到人和你一起工作。如果你的雇主提供推荐奖金，那么招聘对你个人来说是有利可图的。

参加会议

在会议上发言实际上是参加会议的一种很好的方式。你的旅行和费用通常会包括在内。整个周末你都将佩戴一枚演讲者徽章，这会让人们觉得你更酷、更体贴。您将获得演讲者的晚餐，这使得您可以轻松地与其他演讲者联系。我经常发现，和一小群人一起吃一顿丰盛的晚餐，比在喝咖啡休息时间和陌生人随意交谈15分钟，是一种更好的建立联系的方式。

会议分类法

  • 公司内部事件。这可能是在公司会议或工程全体会议上的演示。或者，也许你的公司做内部闪电谈话会议。这些活动是在压力较低的环境中开始公开演讲的好方法。如果你正在演示一个你的团队开发的新产品，那么你将拥有最友好的观众，因为你公司的每个人都希望你的团队做得很棒。
  • 当地聚会。这些往往很低调。气氛非常轻松，有披萨，还有欢迎初级工程师的人群。观众可能很少(< 25人)。这个演讲可能不会上传到YouTube上。这是另一个让你沾水的好地方——最坏的情况是，如果事情很糟糕，你所浪费的只是一次穿越城镇的旅行。
  • 地区会议。主要由当地观众参加的地区性会议往往相对简单。会谈将被录制下来并上传到YouTube上。这一事件可能发生在大学校园里租出去的礼堂和教室里。
  • 头条新闻发布会。这些会议通过将主要技术和地缘政治结构联系在一起来命名，比如JSConfEU。他们会有最高的制作价值，他们的名字会出现在你的简历上，他们发布的关于你演讲的视频会被浏览最多。但是他们也有最高的扬声器标准。

当然，这些都是非常广泛的类别，许多会议可能不会整齐地归入一类。

选择会议

并非所有的会议都是平等的。如果你能在你发言的地方有一些判断力，你应该努力为自己找一个好的会议。要考虑的因素包括:

  1. 会议的主题是什么？也许你可以在DjangoCon上发表关于网页可访问性的演讲，但是如果你对Python不感兴趣，那么会议的其余部分对你来说就没什么价值了。看看往年的日程安排。演讲结束后，你还想参加其他会议吗？有些会议非常好，但可能没有你感兴趣的会议，要么是因为它们是你不感兴趣的技术，要么是因为它们针对的技能水平对你没有用处。如果你是为你的公司招聘，你只需要高级工程师，那么一个内容针对初级受众的会议就不会那么有用。或者如果您需要Ruby开发人员，那么Python会议将不是理想的。
  2. 会议看起来运行良好吗？有些会议比其他会议办得更好。作为一个基本上没有报酬的演讲者，你的时间的一个主要价值是在社区面前看起来不错。如果会议管理不善，可能会偏离这一目标。如果会议组织者在印刷的程序中篡改了你的简历，或者有不可靠的音频/视频设置，这会让你看起来很糟糕。为了评估这一点，我建议看看前几年的视频和会议网站，看看它是否是认真的、注重细节的、有经验的人的作品。
  3. 会议的生产价值是什么？如果你发言是因为你想在互联网上看起来不错，那么不同的会议对这个目标会有不同的影响。在专业舞台上，你会比周末在空荡荡的法学院教室里看起来酷得多。(我不是说这应该是每个人的唯一目标，也不是说你不能在教室里开好会议。)
  4. 你将如何去参加会议？如果活动在你居住地的一日游范围内，参加费用很低。如果你要出国旅行，你应该确定这对你来说是值得的。另外，如果你想把会议旅行和度假结合起来，我强烈建议你在会议结束后去度假，这样你就不会一直为你的演讲感到压力。
  5. 会议在哪里举行？你可能会选择去参加一个会议，因为你想免费去一个城市。这是一个很好的理由，但是如果会议在离市中心45分钟路程的被购物中心包围的会议中心举行，你可能得不到你想要的。在所有的会议活动之间，你可能没有时间真正享受你想去的城市，除非你延长你的行程。
  6. 会议是单轨还是多轨？有些会议一次只有一个演讲，而其他会议则给与会者多种选择。单轨会议可能很好，因为你保证了最大的观众规模。然而，这也意味着听众中可能有人不在乎你的讲话。有了多轨道会议，参加你的演讲的人会对你要说的更感兴趣。但这也带来了焦虑或失望，因为选择你演讲的人没有你想的那么多，你不得不对一小群尴尬的人演讲。多轨会议也可能会给你带来压力，促使你的演讲与你竞争的其他演讲相抗衡。可以问问会议组织者，与会者与发言的比例是多少。会议组织者希望每个人都有一个有趣的演讲，因此他们可能会受到激励，通过安排许多同时进行的演讲来相对分散与会者。你还应该弄清楚你将在哪个房间演讲——是在主题演讲的位置，还是在一个小房间里？你的讲话会被录音吗？有些会议只录制某些曲目。

被录取并发表演讲

关于这一点已经有很多建议了:

  • 说话. io
  • 如何在科技会议上成为更好的演讲者
  • 我们都牛逼！

我就补充几点想法。

许多会议的接受率为10%或更低。为了增加你的胜算，向每个会议提交更多的演讲，并且向许多会议提交。注意:就像你在申请大学一样，召开“安全”和“接触”会议基本上是行不通的，因为录取通知书会源源不断地涌来，在你接受之前，你没有时间去看看你申请的所有地方。相反，我会推荐一个策略，申请你认为有可能被录取的最好的会议，如果你没有被录取，就慢慢降低门槛。

最大的会议可能有偏爱有经验的演讲者的选择过程，因为他们不想在未知的数量上冒险50分钟的时间段。如果你刚刚开始，瞄准较小规模的活动来建立成功会谈的记录会更容易。

如果你被接受在一次会议上做多次演讲，祝贺你！然而，要确保你有足够的精力在彼此接近的情况下进行会谈。而且，即使你有足够的时间来准备两场演讲，仅仅是练习足够让两场50分钟的会议在同一天进行，就可能导致你在到达会议之前就筋疲力尽。如果你是第一次做这两个演讲，这些影响会加剧，所以你需要更多的练习。

如果会议有任何关于他们希望看到的CFP格式的特别说明，或者他们想要的细节，请务必密切关注。对于CFP评分员来说，没有注意细节和遵循指示很容易从拥挤的池中挑选出你的作品。

有什么可以聊的？

如果你想找些话题，这里有几个我见过的被广泛接受的话题:

  • 深入探讨某个专业领域,如可访问性、安全性或性能。你也可以做一个“X代表Y”的框架，比如“开发者的可用性”。
  • 科学项目。这些讲座关注的是一些很酷但没有多少工程师能在日常工作中接触到的东西。例如，我曾经做过一个关于程序内容生成的演讲。
  • 新技术或趋势的介绍。例如，当ES6还是新产品时，你会看到一堆解释其工作原理的演讲。如果你是一个推动这项新技术的人，你的演讲将会得到更多的关注。例如，如果你深入参与React社区，你将有很大的可信度来谈论该生态系统的新发展。
  • 针对常见问题提出解决方案。例如，当一个新的框架变得流行时，对于使用它的代码进行自动化测试的最佳方式，通常会有各种各样的意见。即使你的解决方案不是最好的，如果你是第一个拥有会议视频的人，它也可以成为一个标准。😃

一个常用的策略是为你还没有创造的谈话写一个CFP回应。你会提前几个月拿到录取通知书，这样你就有足够的时间进行实际的谈判。我已经做了五次不同的演讲。每一次，从演讲被接受到我真正拿出足够的内容来证明它是可行的，中间都有一段可怕的时间。但是这种恐惧从来就没有事实根据！😃

另一个重要的考虑因素是:你目前的知识和你要讲的内容之间的差距有多大？当我做我的过程内容生成演讲时，我对相关的计算机科学技术做了大量的研究，并制作了一个演示来说明这些概念的应用。这花了很多时间。相比之下，在我的可访问性谈话中，我只是在谈论我为我的常规工作所做的工作，我不需要制作一个新的演示来展示一些东西。那次谈话比较容易准备。

当我第一次开始演讲时，我依赖于演讲，我必须做更多的学习来展示，因为我在我的职业生涯中更初级，没有太多的东西可以讲。现在我已经看到了更多的东西，我能够产生更接近我日常工作经验的谈话。

金钱角度

如果你以正确的方式看待，会议现场的经济学是有趣的。公司可能会为他们的员工参加会议支付大量的费用，包括机票、每日餐费、会议门票和不花在工作上的时间。员工可以从演讲者那里学到很多东西，但所有的演讲和演示材料通常都可以在网上免费获得。亲临现场的真正附加价值是有机会与其他参与者建立联系，并为公司招聘人才。但是说实话，那些真正为公司招聘的人和那些利用会议寻找新工作的人有什么区别呢？我认为看待一些会议的正确方式不是把它看作是对公司有直接投资回报的努力，而是看作是对员工的额外补贴。当会议在不方便但美丽的地点举行时，比如佛罗里达州的一个岛屿，并且有NodeJS潜艇的完整轨道，perk角变得更加清晰。不是说这有什么错，要说清楚！

扬声器方面的经济学也很有趣。除了一张免费的会议入场券之外，大多数演讲者都不收取演讲费。一些会议会报销差旅费，尽管大多数会很乐意让演讲者的雇主来支付。如果你接受了演讲的邀请，你应该清楚你从交流中获得了价值。除了可能的旅行报道，你还会得到无形的好处，比如公众的关注度，你简历上的另一行项目，以及看着别人听你说一个小时的话所带来的内在快乐😃。这对你来说值得吗？

我粗略估计我可以花40个小时准备和练习一次演讲。如果一个会议为我的演讲支付600美元的差旅费+餐费+礼品费，那么就是15美元/小时，也就是每年30，000美元。据估计，作为一名Lyft司机，你可以获得更高的时薪，压力也更小。在会议上发言并不全是为了钱，但你应该清楚自己还能得到哪些好处，让你觉得值得。

有些会议是营利性的。其他人为组织者赚很少甚至没有钱，并且在很大程度上是对社区的热爱。一些营利性会议会伪装成后者。在你接受之前，确保你知道会议是什么，以及你是否对它感到满意。有人给你一个演讲的机会，实际上是要求你为他们的营利性活动提供免费内容吗？如果是这样，你能接受吗？(如果是也可以。你应该知道它是什么。)

说到差旅费报销，在你接受组织者的提议之前，和他们就报销额度达成一致。这对于国际旅行来说更为重要。例如，你可能有机会在巴黎演讲，但组织者可能只愿意支付如此痛苦的飞行行程，以至于你宁愿不做。在做出承诺和开始准备谈话之前，提前想好这一点。

结论

人们应该参加符合他们目标的会议。然而，我的建议是精明地找到合适的人选，这并不是说会议是浪费时间。当人们参加适合他们的会议并在会上发言时，这可能会是一次奇妙的经历，为工作提供新的见解，并与社区建立持久的联系。

如果你喜欢这个，请鼓掌，让其他人也能发现。感谢阅读！
