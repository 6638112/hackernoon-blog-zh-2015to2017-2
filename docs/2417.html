我对送餐业务的定位如下:

使客户能够与附近的其他客户一起以折扣价进行按需团体订购

想象一下，每次打开应用程序，你都会对不同的交易感到惊讶。几分钟后就消失的交易。

可变折扣将是食品订单的奖励部分。不像目前持续数天的静态应用内促销，而是根据当时你周围的人和他们点了什么来随时改变交易。

作为客户，您可以控制自己可以享受的折扣。

对于那些对这种功能的机制感兴趣的人，我在下面通过模型和工作原理详细介绍了关键方面。

概念描述

  1. 任何客户(“发起方”)均可发起集团订单
  2. 为了在食品交付应用程序上发布团体订单，发起人从餐馆中选择一餐，并设置下订单的时间范围(有效期)
  3. 在有效期内，附近位置的其他客户(邻居)可以在订单到期前加入订单。加入的邻居数量越多，该组所有客户的折扣就越大
  4. 如果在到期前没有邻居加入组订单，则订单不会获得折扣，订单将被视为正常订单
  5. 一名司机从餐馆收取所有订单(同一团队订单的一部分),并将其交付给邻居——他们都位于相同/附近的位置

UberEats和Postmates已经通过匹配算法在司机、餐馆和顾客之间实现了运输物流。这一概念进一步优化了运营，并将形成一个更强大的社区。

对于那些知道Nir Eyal的“挂钩画布”的人来说，上面的步骤(1)和(3)就是“投资”。另一项投资可能是客户之间的距离越来越近，从而引发更好的交易。

工作原理

折扣是如何运作的？

它是两个部分的总和:

  • 餐厅折扣("休息。Disct"):

餐厅折扣阈值是与餐厅预先约定的，并且是该餐厅的每组订单的邻居数量和/或特定膳食的的函数(例如，“从餐厅X订购任何膳食的3个邻居为5%，从餐厅X订购夏威夷扑克碗的4个邻居为10%)

理由 : ↗的收入、↗的员工利用率和↗的规模经济(同一餐的许多订单)

  • 送货折扣(" Deliv。Disct"):

交付折扣阈值是每组订单的邻居数量的函数(即，交付成本的减少)，并且取决于要优先考虑的财务指标:例如，利润(最小。德利夫。Disct，只有休息。Disct)、收入(最大。德利夫。关于休息的讨论。吸引音量)

基本原理 : ↗收入，↘成本(↘平均。#司机/订单)，解决高峰时间问题(高效交付)

→发起者创建团体订单后，对折扣的估计(其余。Disct + Deliv。Disct)来指示每个潜在的附加邻居。加盟邻居也能看到这些估计。

→一旦下了团体订单(有效期结束)，就会计算出最终适用的折扣。该组订单的所有邻居受益于相同的最终折扣。

为什么每个团体订单都有截止日期？

目的是让这些团体订单在高峰时段特别激活。如果在高峰时段提供5-20%的折扣，下订单的客户可能会乐意多等10-30分钟。

额外的10-30分钟是必要的时间，用于(I)团体订单达到其有效期的末尾，以及(ii)将饭菜送到队列前面的邻居手中(例如，先到先得)。例如，发起者可以通过首先被传递来获得奖励。

我们人类喜欢过期的礼物，这让我们觉得我们活在当下。有有效期有两个好处:(1)有限的等待时间，和(2)创造一种紧迫感。

是什么让这成为可能？

顾客年收入

超过40–45%的UberEats和Postmates客户年收入低于5万美元。此外，20–30%的客户每年收入高达10万美元。假设大多数顾客对价格敏感是公平的，因此团体订单会吸引他们，因为它允许他们以较低的价格就餐。

顾客区

大多数用户从密度较高的城市地区订购，使得陌生人之间的团体订单变得可行。需要进一步获得关于客户密度的额外信息(例如，位于200米半径范围内的客户数量)，以充分确认这一点。

挑剔及其与顾客年龄的联系

假设:年龄在某种程度上与顾客的挑剔(对食物选择的挑剔)有关:

  • 年轻用户(40-50%的客户群)通常更愿意妥协，并且更容易加入团体订单，即使它没有为他们提供他们喜欢的餐饮选择/餐厅选择
  • 中年客户(40–45%的客户群)预计会更有选择性，但仍愿意不时妥协

即使大多数客户预计不会太挑剔，UberEats和Postmates也会收集数据，以更好地了解他们的偏好。一旦团体订单符合客户的偏好，就会发送通知，邀请客户加入订单。

在应用程序上点餐花费的平均时间很低，这表明顾客专注于满足他们的食欲，而不是出于快乐而吃东西。团体订单将使快速订购变得更加容易。

技术基础设施

优步的Marketplace和Postmates解决方案已经将真实世界的实时请求和位置导入到他们的技术基础设施中。

他们的系统已经实时处理来自顾客、餐馆和司机的pings，以匹配他们。客户(邻居)之间只需要一个额外的“匹配”机制，集团订单就能成为现实。

UberEats和Postmates的挑战

  • UberEats和Postmates已经能够依靠外部资金来补贴他们的单位经济。他们现在需要获得无与伦比的客户体验，才能成为首选。
  • UberEats和Postmates被认为是的食品配送服务，他们别无选择，只能在任何特定时间满足每位顾客的需求。我们的社会规范迫使我们在预定的时间吃一日三餐。这造成了高峰时间的交付挑战
  • 从优步到UberEats的激增定价转换是次优的，因为客户有更多的选择(家庭烹饪、超市送货、超市实物购买、餐馆内就餐、非UberEats食品配送)和更高的时间灵活性。此外，不向顾客收取任何费用也违背了优步的定价模式(相反，它从餐厅收取25-30%的费用，而竞争对手收取12-23%的费用)

E.限制

交付限制:

  • 由于每位司机只能运送有限数量的餐食，因此每份团体订单的餐食数量应受到汽车/摩托车可用空间的限制
  • 总折扣的计算应考虑到这一点:例如，对于非常大的订单(对于餐厅来说很大，对于一个司机来说太大):高餐厅折扣，低送货折扣

密度和时间:

  • 密度需要足够高，才能使它发挥作用，也就是说，饭菜要在有限的时间内送到每个邻居手中
  • 每个显示的团体订单需要满足两个标准:(I)餐馆将外卖送到顾客的位置，以及(ii)顾客位于发起者附近。“邻近”将由UberEats / Postmates根据其交付指标确定:是否是同一栋建筑，在3个街区内，取决于交通状况等
