系统故障。我为什么买闲置股票。

原文：https://medium.com/hackernoon/glitch-in-the-system-why-im-buying-slack-stock-f67fd01599e8?source=collection_archive---------11-----------------------#2017-04-26

我已经开始了新的一章，我想分享我的知识、经历、想法，谈论我感兴趣的事情。我想测试我的写作能力，以深入的“保持100分”现实生活内容为特色。我每两天写一篇新专栏，持续30天。主题包括科技、健康、旅游、食品、创业、创新、流行文化、体育、自行车、汽车和风险投资。请随意插话。第一条。拥抱和亲吻。

我记得911袭击美国时我在哪里。毕竟，谁能忘记那样的事情。我还记得当特朗普成为特朗普总统时，我在哪里，看着从东海岸到西海岸的选票，听着专家向美国人民保证他们的预测是准确的。在工作中，在那些“被车灯照着的鹿”的时间里，几乎没有什么成就。有影响的事件通常会留在我们的记忆中，偶尔我们会想起这些事件，我们的大脑会发现让我们及时重温那一刻很简单。我还记得当我公司的一个人建议我们尝试这种叫做 Slack 的新信息服务时，我在哪里。

Slack有一个现在已经广为人知的故事，讲述了斯图尔特·巴特菲尔德与他在Flickr共事的一些核心团队一起创办一家名为T2的游戏公司的失败尝试。他们在TinySpeck的第一个产品叫做Glitch，这是一个异想天开的在线游戏，没有足够的玩家继续运行，最终关闭。这并不意味着他们没有任何资金或不知道他们在做什么。TinySpeck在2009年获得了150万美元的天使投资，并在2010年获得了来自 Accel 和 Andreessen Horowitz 的500万美元的首轮融资。两个最大最活跃的投资公司。Glitch原定于2011年3月推出，但像所有事情一样，它需要两倍的时间和两倍的努力。Glitch于2011年9月短暂推出，但很快被撤回以改善总体游戏性。虽然Glitch在接下来的几个月里再次发布，而且这款游戏有一小群狂热的追随者，但有时最好是减少损失，继续前进。

在TinySpeck构建Glitch时，开发团队在对Yammer和当时可用的其他通信工具感到失望后，需要一个内部通信工具。由于有时远程工作，开发团队在流行的聊天系统IRC上构建了他们自己的通信工具，以保持彼此在循环中。团队花在彼此聊天上的时间几乎和他们花在开发上的时间一样多。看到一些值得探索的东西，巴特菲尔德在2012年11月关闭了Glitch，并将TinySpeck转变为Slack Technologies。6个月后，Slack推出了一款团队沟通产品，其电梯式宣传是:“将所有团队沟通集中到一个地方，并使其可搜索”。

这个故事是创业梦想的来源。

自2013年成立以来，Slack已经重建了他们远离IRC的技术堆栈，并成功成为最大和最聪明的技术公司之一。

Slack成为Tappr的官方产品

当我们在Tappr正式开始使用Slack时，已经是2015年2月，也就是Slack推出后的将近18个月。在此之前，我们尝试了几款Atlassian的产品，包括吉拉、HipChat和Confluence以及Skype，用于追踪事物和交流。当时，统一使用Atlassian的工具感觉就像是彼此断开连接一样。后来我研究到，实际上HipChat和Confluence是Atlassian收购的产品，而不是从头开始构建的，所以我的直觉是正确的。虽然Atlassian的一系列产品都很棒，但它们放在一起是不是也很棒？当然是改天的话题了。

在Tappr，以真正的初创企业的方式(在12秒或更短的时间内)做出决定，创建一个Slack账户，并将我们所有的通信转移到Slack，而不是电子邮件、Atlassian或Skype。与其他更早采用Slack的公司相比，你可以认为我们的变化是后期或落后的状态，但在澳大利亚听到大事情确实需要一些时间。

顺便说一句， 我很自豪地说，2006年的一个暑假，我去了美国，我是第一批离开大学生开始使用脸书的用户之一，所以这就是我的早期用户“名声”。

当我们在2015年初在Tappr将Slack作为我们的主要沟通工具时， Slack 已经以11亿美元的估值筹集了1 . 2亿美元。Slack的联合创始人是 TechCrunch的“年度Crunchie”，拥有70，000名付费客户，每月收入增长到100万美元！这不仅仅是令人印象深刻，这是令人发指的。

Slack要成功需要什么条件？

互联网是个奇妙的东西，尤其是在研究方面。

2005年5月，谷歌实验室部门尝试了一个名为“谷歌骑手”的项目。根据在线文档，该项目详细介绍了“当你需要找到出租车、豪华轿车或班车服务时的拼车服务”。如果你和我想的一样，这听起来确实和T2优步T3的基础相似。2009年，这个项目脱离了电网，再也没有回来。这是作为谷歌地图页面一部分的小小的维基百科条目。决定放弃谷歌骑手搜索服务有很多因素，但最重要的是，技术还不具备成功推出这项服务的可行性。一晃几个月过去了，优步于2009年成立，直到2011年才正式推出用户如今所熟知的技术。从2009年到2011年的两年间，一些事件改变了这一格局:

  • 苹果的iPhone 4于2010年6月24日推出，除了修复糟糕的电池寿命之外，最大的改进之一是对“A-GPS”或辅助GPS的改进，它不仅使用GPS在快速时间内协调iPhone的位置，还使用蜂窝塔网络进行三角定位。在2009-2011年间，智能手机上的GPS精度从大约25+米上升到8米。自2011年以来，在覆盖良好的地区，iPhones下降了几英尺。
  • 2009年，随着高容量网络的出现、计算机和存储设备成本的降低以及面向服务架构的广泛采用，云计算得到了广泛采用。

除了让一家公司专注于像优步这样的服务之外，这项技术还处于一个关键点，与现有的出租车或豪华轿车公司或过去时代的其他科技公司相比，它能够提供更好的服务。优步的部分成功可能来自技术的可获得性和可负担性，以及社会行为中不断变化的公众观念。

当Slack在2013年从TinySpeck转型时，云和移动计算动摇了软件安全和存储的旧模式。随着存储容量的增加，移动设备变得越来越快，像 Dropbox 、 Salesforce 、 Evernote 以及一大批其他生产力和专业应用程序和服务可用，Slack被恰当地定位为蓬勃发展。以及向云计算的转变，对网络营销的关注，生态系统的发展为Slack将其他服务整合到他们的产品中铺平了道路。理解这种从“构建一切”式思维到“让我们整合”的动态变化是Slack受益最多的地方，并为其他公司现在复制铺平了道路。

现在，供服务和平台合作伙伴集成的公共API或SDK已经成为标准。当你与专注于特定领域的公司整合，并利用他们的网络来创造销售渠道时，为什么要花费时间、精力和数百万美元来建立一切呢？

我曾经见过一位苹果高管，当我们讨论Tappr的未来愿景时，我问苹果希望进一步探索的下一个大趋势是什么。他认为“生态系统和市场”是市场的发展趋势。

【Slack擅长什么

Slack并不是第一个开发消息通讯工具的公司，Skype和其他公司早就将此作为他们产品的主要特色。然而，快速的速度，能够搜索以前的消息，轻松地添加用户，不仅将其他服务集成到Slack中，还可以在没有计算机科学学位的情况下添加自己的网络挂钩，这些组合是快速采用的关键。一体化的世界是一个更美好的世界。

有一段时间，免费增值模式让用户进入了大门，这种模式不应该被Slack的成功所忽视，但事实是，他们最近打开了他们的订阅服务，免费用户愿意付费，这说明了他们用户的粘性和服务的价值。

我不喜欢头衔，但在公司生命的某个时刻，我可以看到，拥有一个专注的CMO(首席营销官)是有益的。典型的CMO角色是风险投资人会推动的，因为不了解销售过程前端的基本原理，风险投资人就会紧张。当Slack的估值达到10亿美元时，他们才刚刚开始寻找CMO。他们在他们的“请求邀请”病毒式公关营销策略中富有创造性和足智多谋，在第一天有8000人注册，在两周内，这个数字增长到了15000人。

产品中经常被忽略的一个方面是告诉你的用户为什么他们需要你的产品，这也是Slack在收到最初的用户反馈时所关注的。设计师通常会建议，如果你必须教别人使用你的产品，那么你就是一个失败的设计师。我在某种程度上同意，但是如果你的产品从根本上改变了行为，而不是改善了现有的行为，那么可能需要进行可能性和行为改变的教育。在“测试”阶段，Slack学到了很多东西，但向用户、公司和第三方开发者传授该产品的功能对于用户的早期采用至关重要。

直到优步提供了一种无摩擦且物超所值的服务，我才知道我对出租车行业有意见。我对Slack也有同样的感觉，直到Slack给我指明了方向，我才知道自己在与其他团队成员进行深入沟通方面存在问题。

Slack对我们来说太好了，我们无法抗拒，因此我们更喜欢使用它，而不是我们所能使用的其他工具。

渠道:我们有多少渠道？

当我们开始在Tappr使用Slack时，我们在标准频道上有5个左右的频道，如#通用或#随机。然而，很快我们就意识到团队中的每个人都在一个频道里讨论所有的事情。因此，我们开始为开发、运营、销售和营销创造新的渠道。我们认为这是一个好的开始，并对结果感到满意。

随着时间的推移，这些频道中的每一个都需要分成多个更集中的频道。开发变成了 #dev_hardware 、 #dev_api 、 #dev_tech_leads 和 #delivery_manager 。运营、客户支持、营销和业务发展也出现了类似的扩张。我们甚至尝试让电子邮件进入我们的直接信息渠道，看看我们是否可以完全没有电子邮件。

对于每个渠道，我们添加了新的集成或网络挂钩，这给团队的一些人带来了很多噪音，而其他人则发现新提供的信息很有见地。你可以想象，我们再次分离了渠道，让员工选择最适合他们的。

在这个过程的最后，我们有65个频道，这还不包括私人频道和存档频道。经过6个月的使用，它太多了，我们开始挑选可以合并或存档的频道。截至2017年复活节，我们减少到45个频道。

值得讨论的一个方面是通过Slack创建的内容。作为Tappr为AusIndustry提供的年度研发(R&D)资助的一部分，我们提供了来自某些Slack渠道的数据作为提交的一部分。我们发现，当讨论我们如何在R&D活动中克服可能的未知时，我们发现我们为什么做出某些决定的整个过程在Slack中有很好的记录，尽管事实上也有数百个表情符号。Dev的将会是dev的，我确信AusIndustry的人会从中得到乐趣。

自定义表情符号——slack mojis

我花了一段时间才赶上表情符号的潮流，但现在我已经稳稳地站在了表情符号这座大山的顶端，不会很快下山。Slack的一个鲜为人知的功能是创建或添加自定义表情符号的能力。Slack自带了大量的默认表情符号。然而，当披萨日只有一个旋转的披萨表情符号时，你不会失望的。

光明的机器人集成未来

你可以为Slack对第三方开发者社区参与度的惊人提升建立一个案例。早期，Slack与 Google Docs 、 Heroku 、 GitHub 、 ZenDesk 合作，这种早期的整合为其他第三方开发者将他们的服务与Slack整合铺平了道路。

之前，我用Google Cal查看我的日历，用ZenDesk查看客户服务单，用对讲机和客户聊天，用Git查看工作效率，用电子邮件给同事发邮件。更不用说，我们在API中添加了几个web挂钩来查看所有销售和交易。

我现在只查看Slack的内部信息，偶尔查看电子邮件来管理我的初创公司。

Slack和其他科技公司之间的一个显著区别是，Slack本质上是一个沟通工具，你可以使用Slack直接与其他开发者聊天，或者更重要的是，与Slack开发团队聊天。你可以看到并参与Slack开发团队 Trello 的董事会，了解他们正在做什么，查看他们的路线图，并做出贡献。

现在，2017年有超过750个应用可用，Slack的开发社区非常强大，并且仍在增长。很多这样的整合，我知道我需要或者能够增加价值。在开始使用之前，我并不知道我们需要将ZenDesk集成到一个聊天工具中。如今，我们一直在Slack中测试值得注意的集成，这是Slack魅力的一部分；看看我们使用的服务中有多少可以集成到Slack中。

我最喜欢的包括:

  • 泰特拉 : 喜欢维基的？这个维基帮助你的团队在内部组织和分享重要的知识。
  • Zeplin:面向设计师和工程师的协作app。本质上是一个关注UI的流畅风格指南。
  • birthday bot:不确定某人的生日？永远不要错过队友的生日。通过私人暗示和生日问候，BirthdayBot将团队聚集在一起。

下一个前沿:聊天机器人

最近有很多关于科技未来发展的趋势，其中一个经常谈论的话题就是聊天机器人。我以前用过一些聊天机器人，我并不怀疑它们的实用性，只是怀疑它们是否会流行起来。

我为同事卡洛斯·德卡瓦略做了一些用户体验分析，他开发了 Go-Brisbane应用。在2009年至2010年的巅峰时期，Go-Brisbane有40万次下载，但你对交通应用的热情是有限的，尤其是如果它还没有支付账单的话。随着时间的推移，这款应用没有得到太多关注，用户数量也开始慢慢减少。我的一个建议是注册后第一次打开一个对话聊天机器人。不像其他聊天机器人只是等待你做一些事情，一个正确实现的对话聊天机器人可以触发应用程序的主要动作，并为用户节省大量的时间和精力。

由于我对新事物感兴趣，我还在学习德语，因为我六月份要去柏林。我下载了 多林哥 并试用了一下。DuoLingo相当简单易用，但是你确实需要每天使用它来获得最大的影响。我发现自己可能一周使用两次，并且记得把它作为我一天的一部分来使用，这需要一些时间来适应。最近，DuoLingo增加了一个德国聊天机器人作为学习练习的一部分。现在DuoLingo(通过聊天机器人)像Slack一样和我说话。

增加一个熟悉的行为，比如早上和我聊天，改变了我对DuoLingo应用的使用，Slack目前有许多类似设计的聊天机器人正在使用或开发中。聊天机器人几乎可以用于任何事情，Slack处于利用聊天机器人运动的绝佳位置。

脸书式上瘾——凡事都用Slack】

我已经大到可以记得MSN聊天了，我还记得过了睡觉时间还熬夜和朋友、家人和其他人聊天。我记得我对脸书有同样的瘾，我想我对懒散也有类似的瘾…问我女朋友。与脸书甚至Whatsapp不同的是，我发现Slack的罪恶感更少，效率更高。

根据Slack的内部分析，许多Slack用户平均每天有10个小时与这项服务保持联系。我一天不睡10个小时！这是一个惊人的统计数字。

当你对一个新功能的进展有疑问时，你只需简单地问一下。你知道迈克尔或阿尔文做了一些与你的问题相关的工作，所以你会“提到他们，并确保他们都看到了。

如果我们发现了一个bug，你只需在其中一个频道中提到它，并期望它会被处理掉。然后你可以看到一个开发人员创建了这个bug，开始处理它，一旦解决了，Slack就会告诉我。毕竟，频道里有各种各样的人，所以肯定有人会为此做些什么。

所有这些互动都将在Slack中进行，而在此之前，它们将通过电子邮件、面对面交流或通过其他软件订阅来进行。有像bug追踪器或wiki这样的工具，但没有这么有趣，我不得不去别的地方做。有了Slack，都在一个地方了。

使用Slack，尤其是在一家科技公司，你有太多的活动，而且使用起来很有趣，这促使我们关注它。

回馈社会

2015年末，Slack与几家风险投资伙伴合作，启动了一只8000万美元的投资基金，专注于投资补充Slack核心技术的软件项目。

这不是科技公司第一次用自己的风险基金投资自己的网络，但毫无疑问，它们是最活跃的公司之一。在来自 Tech Crunch 的一篇文章中，Slack声称已经在一系列补充服务领域总共投资了25笔，其中包括 StatsBot 、 SwayFinance 、 Guru 和 DataFox 等等。

警告:需要平衡

虽然Slack改变了Tappr内部沟通的方式，但它可能会很吵，让人分心，更不用说没有人情味了。

我们已经尝试了几乎所有的bot或集成，但每天可以使用的1001个CatFacts 的数量是有限制的。我们甚至尝试了站立机器人，看看我们是否可以松弛我们的开发冲刺，然而，试图讨论复杂的主题可能很难使用松弛，并且不会促进团队建设。

在开发中采用更传统的敏捷方面有助于更多的面对面会议，并且通常可以在更短的时间内解决更复杂的问题。在我们的空闲高峰时期，我们有时会通过Slack与多人就复杂的问题聊天长达20分钟，而面对面的会议本可以在2分钟内解决这些问题。

简而言之，有效的沟通需要的不仅仅是惊人的连通性。许多Tappr团队成员抱怨说，他们不知道公司发生了什么，也不知道为什么会做出某些决定。执行管理团队(EMT)确保每两周召开一次会议(有啤酒、葡萄酒和小吃),供所有团队成员与团队领导或在场的其他员工一起提问。EMT详细介绍了公司的进展、里程碑和当前进展。任何人都可以问任何问题，任何话题都不是禁忌。

我听说过其他一些公司，他们将Slack作为唯一的沟通工具，并发现了类似的爱情故事，但在严重依赖后，他们仍然称赞适度。我听说过一些公司暂停使用Slack，因为它可能会很吵或者没有人情味，我理解为什么，毕竟团队精神和理智很重要，不管成年人如何正确使用，Slack无疑是初创公司的必备工具。

我在买闲置的股票。

毫无疑问，Slack是一项吸引人的服务，任何公司都可以把它作为交流的枢纽。当然，有些人担心懈怠会接管你的整个生活，但我们这里不是在谈论天网。在Tappr，我们沉迷于使用它做任何事情，然而随着时间的推移，常识占了上风，我们适度地使用Slack。
